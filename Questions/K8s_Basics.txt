# 🔰 1. What is Kubernetes?

Kubernetes (often called **K8s**) is a **container orchestration platform**.

👉 In simple terms:

* Docker runs containers
* Kubernetes **manages containers at scale**

It handles:

* Deployment
* Scaling
* Load balancing
* Self-healing (restarts failed containers)

---

# ⚙️ 2. Why Kubernetes in AWS DevOps?

In AWS, Kubernetes is mainly used through:

👉 **Amazon EKS**

Why teams prefer it:

* Managed control plane (less maintenance)
* Easy integration with AWS services (IAM, VPC, ALB)
* Production-ready

---

# 🏗️ 3. Kubernetes Architecture (Must Know)

Think of Kubernetes like a **master-worker system**.

### 🔹 Control Plane (Master)

Manages the cluster:

* API Server → entry point
* Scheduler → assigns workloads
* Controller Manager → maintains desired state

### 🔹 Worker Nodes

Where your apps run:

* Kubelet → talks to master
* Container Runtime → runs containers (Docker/containerd)

---

# 📦 4. Core Kubernetes Objects (Very Important)

These are your daily DevOps tools:

### 🔹 Pod (smallest unit)

* A pod = 1 or more containers
* Usually 1 container per pod

### 🔹 Deployment

* Manages pods
* Supports scaling & rolling updates

### 🔹 Service

* Exposes your app
* Types:

  * ClusterIP (internal)
  * NodePort
  * LoadBalancer (used in AWS)

### 🔹 Namespace

* Logical separation (dev, prod, etc.)

---

# 🧾 5. Basic Example (Deployment + Service)

Here’s a simple YAML:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx
```

Service:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
```

# 🖥️ 6. kubectl Commands (Daily Use)

These are like your bread and butter:

```bash
kubectl get pods
kubectl get nodes
kubectl get svc
kubectl describe pod <name>
kubectl logs <pod>
kubectl apply -f file.yaml
kubectl delete -f file.yaml
```

# ☁️ 7. Kubernetes in AWS (Real DevOps Flow)

Typical workflow:

1. Create cluster using:

   * eksctl / Terraform

2. Configure access:

   ```bash
   aws eks update-kubeconfig
   ```

3. Deploy app:

   ```bash
   kubectl apply -f deployment.yaml
   ```

4. Expose using:

   * LoadBalancer (creates AWS ELB)
